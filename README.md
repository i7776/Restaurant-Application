# Restaurant Management System (RMS)

## О проекте

**Restaurant Management System** — это десктопное приложение, разработанное в 2025 году для автоматизации управления ресторанным бизнесом. Проект представляет собой законченное решение для ресторанов, сетей заведений или отдельных точек общепита. Система позволяет автоматизировать ключевые бизнес-процессы: от управления персоналом и меню до контроля заказов и поставок.

Ключевая особенность — строгая ролевая модель, которая разграничивает доступ для владельцев, администраторов, официантов, поваров и барменов, обеспечивая безопасность данных и удобство работы каждого сотрудника.

## Функциональные возможности

*   **Ролевая модель и авторизация:**
    *   Вход в систему с проверкой учетных данных (хэширование паролей через BCrypt).
    *   Пять ролей: Владелец, Администратор, Официант, Повар, Бармен.
    *   Динамическая настройка интерфейса в зависимости от прав пользователя.
*   **Управление персоналом:**
    *   Создание, просмотр и удаление сотрудников.
    *   Привязка сотрудников к конкретному ресторану.
    *   Контроль прав: владелец управляет только сотрудниками своих ресторанов.
*   **Управление меню:**
    *   CRUD-операции для блюд (название, описание, цена).
    *   Проверка уникальности блюда в рамках одного ресторана.
    *   Защита от удаления блюд, которые фигурируют в заказах.
*   **Управление заказами:**
    *   Создание заказа с выбором блюд из меню и автоматическим подсчетом суммы.
    *   Изменение статуса заказа по цепочке: "Новый" → "Принят" → "Готовится" → "Готов к выдаче" → "Завершен".
    *   Фильтрация активных заказов по ресторану и статусу.
*   **Управление поставками** (базовый функционал).
*   **Дашборд владельца** для агрегированной информации по ресторану.

## Стек технологий

*   **Язык и платформа:** C#, .NET 8.0 
*   **Интерфейс:** Windows Forms (с кастомизацией под фирменный стиль)
*   **База данных:** Microsoft SQL Server Express / LocalDB
*   **ORM:** Entity Framework Core (Code First, Migrations)
*   **Безопасность:** BCrypt для хэширования паролей
*   **Архитектура:** Многоуровневая (Data Access Layer, бизнес-логика, представление)

##  Архитектура приложения

Проект построен по принципу разделения ответственности для обеспечения масштабируемости и простоты поддержки.

## Структура базы данных

Основные сущности базы данных `RestaurantDB_1`:

*   **Users (ApplicationUser):** Хранит данные сотрудников, хэши паролей и связь с ролями и рестораном.
*   **Roles:** Справочник ролей с описанием прав.
*   **Restaurants:** Информация о заведениях (адрес, контакты, тип кухни).
*   **MenuItems:** Блюда, привязанные к конкретному ресторану.
*   **Orders & OrderDetails:** Заказы и их детализация (связь с блюдами и ценами).
*   **Supplies:** Поставки продуктов/товаров.

Связи построены на внешних ключах для обеспечения целостности данных (каскадное удаление настроено с учетом бизнес-логики).

## Ролевая модель и доступ

Система гибко настраивает интерфейс для каждой роли:

*   **Владелец:** Полный доступ ко всем ресторанам своей сети, управление персоналом, меню, заказами, поставками и дашборд с аналитикой.
*   **Администратор:** Управление пользователями (кроме владельцев), настройка ресторанов, контроль заказов.
*   **Официант:** Работа с заказами (создание, просмотр активных, изменение статуса до "Завершен").
*   **Повар:** Видит заказы, требующие приготовления, меняет статус на "Готовится" и "Готов к выдаче".
*   **Бармен:** Аналогично повару, но фокус на напитках (в текущей реализации — общий поток заказов с разграничением по ролям).

>  **Безопасность:** Проверка прав доступа происходит как на уровне интерфейса (скрытие кнопок), так и на уровне бизнес-логики, что исключает возможность несанкционированных действий.

## Особенности интерфейса (UI/UX)

*   **Единый стиль:** Все формы выполнены в единой цветовой гамме (синий оттенок `#3c6799`) и шрифте Segoe UI.
*   **Адаптивность под роль:** Интерфейс "подстраивается" под пользователя, показывая только нужный функционал.
*   **Обратная связь:** Интуитивные сообщения об ошибках и подтверждения действий (например, перед удалением).
